## Задание 1. Заполнение двумерных массивов

### Условие:
1. Первый массив размерностью n × m заполняется данными, вводимыми с клавиатуры (заполнение по строкам от последних элементов к первым)
2. Второй массив размерностью n × n заполняется случайными числами:
   - Элементы ниже главной диагонали: [-17; 36]
   - Элементы на и выше главной диагонали: [100; 10000]
3. Третий массив размерностью n × n заполняется по образцу (нижняя треугольная матрица)

### Алгоритм решения:
- Создан класс `Matrix` с несколькими конструкторами
- Конструктор 1: цикл по строкам, внутри цикл по столбцам от m-1 до 0, ввод с клавиатуры
- Конструктор 2: проверка позиции элемента (j < i - ниже диагонали, иначе - на и выше)
- Конструктор 3: заполнение по столбцам, начиная с диагонали, последовательными числами

### Тесты:

**Тест 1.1: Ввод с клавиатуры матрицы 2×3**
```
Ввод:
Строка 1: 1 2 3
Строка 2: 4 5 6

Результат:
     1     2     3
     4     5     6
```
✓ Элементы заполнены корректно

**Тест 1.2: Случайное заполнение матрицы 3×3**
```
Результат :
   156   8234   5421
    -5    234   9876
   -12     -3    156
```


**Тест 1.3: Нижняя треугольная матрица 5×5**
```
Результат:
     1     0     0     0     0
     2     3     0     0     0
     4     5     6     0     0
     7     8     9    10     0
    11    12    13    14    15
```

## Задание 2. Поиск максимального неповторяющегося элемента

### Условие:
Найти максимальный из неповторяющихся (встречающихся только один раз) элементов двумерного массива

### Алгоритм решения:
1. Создан словарь `Dictionary<int, int>` для подсчета частоты каждого элемента
2. Первый проход по матрице: подсчет количества вхождений каждого числа
3. Второй проход по словарю: поиск максимального ключа со значением 1
4. Возврат nullable int (null если все элементы повторяются)

### Тесты:

**Тест 2.1: Матрица с неповторяющимися элементами**
```
Входные данные:
     1     2     3
     4     2     5
     6     7     3

Неповторяющиеся: 1, 4, 5, 6, 7
Повторяющиеся: 2 (2 раза), 3 (2 раза)

Ожидаемый результат: 7
Полученный результат: 7
```
✓ Тест пройден

**Тест 2.2: Все элементы повторяются**
```
Входные данные:
     1     1     2
     2     3     3
     4     4     5

Результат: Нет неповторяющихся элементов
```
✓ Возвращается null

**Тест 2.3: Случайная матрица 3×3 [0;10]**
```
Входные данные (пример):
     5     3     8
     2     5     1
     9     3     7

Неповторяющиеся: 8, 2, 1, 9, 7
Результат: 9
```
✓ Тест пройден

## Задание 3. Матричные операции

### Условие:
Вычислить значение матричного выражения 2*A - 3*B*C с использованием перегрузки операторов

### Алгоритм решения:
- Перегружены операторы: `+`, `-`, `*` (матрица × матрица), `*` (скаляр × матрица)
- Сложение/вычитание: поэлементная операция с проверкой размеров
- Умножение матриц: тройной цикл (i, j, k) с суммой произведений
- Умножение на скаляр: каждый элемент умножается на число
- Перегружен метод `ToString()` для красивого вывода матрицы

### Тесты:

**Тест 3.1: Вычисление 2*A - 3*B*C
```
Входные данные:
A = [1 2]    B = [2 0]    C = [1 1]
    [3 4]        [1 2]        [0 1]

Шаг 1: B*C = [2*1+0*0   2*1+0*1] = [2  2]
             [1*1+2*0   1*1+2*1]   [1  3]

Шаг 2: 3*B*C = [6  6]
               [3  9]

Шаг 3: 2*A = [2  4]
             [6  8]

Шаг 4: 2*A - 3*B*C = [2-6   4-6] = [-4  -2]
                       [6-3   8-9]   [ 3  -1]

Ожидаемый результат:
    -4    -2
     3    -1
```

## Задание 6. Работа с текстовым файлом (один элемент в строке)

### Условие:
Файл содержит целые числа (по одному в строке). Количество элементов чётное. Определить разность суммы элементов первой и второй половины файла

### Алгоритм решения:
1. Чтение всех чисел из файла в список `List<int>`
2. Проверка на чётность количества элементов
3. Деление списка пополам (mid = count / 2)
4. Вычисление суммы первой половины (0 to mid-1)
5. Вычисление суммы второй половины (mid to end)
6. Возврат разности (sumFirst - sumSecond)

### Тесты:

**Тест 6.1: 10 чисел в файле**
```
Содержимое файла task6.txt:
10
20
30
40
50
-10
-20
-30
-40
-50

Первая половина: 10+20+30+40+50 = 150
Вторая половина: (-10)+(-20)+(-30)+(-40)+(-50) = -150
Разность: 150 - (-150) = 300

Результат программы: 300
```
✓ Тест пройден

**Тест 6.2: 6 чисел**
```
Содержимое файла:
5
5
5
1
1
1

Первая половина: 5+5+5 = 15
Вторая половина: 1+1+1 = 3
Разность: 15 - 3 = 12

Результат программы: 12
```
✓ Тест пройден

## Задание 8. Работа с текстовым файлом (строки)

### Условие:
Переписать в другой текстовый файл самую короткую и самую длинную строки

### Алгоритм решения:
1. Чтение всех непустых строк из исходного файла в список
2. Инициализация shortest и longest первой строкой
3. Проход по всем строкам:
   - Если длина строки < shortest.Length → обновление shortest
   - Если длина строки > longest.Length → обновление longest
4. Запись результатов в новый файл с заголовками

### Тесты:

**Тест 8.1: Файл с 5 строками разной длины**
```
Содержимое task8_source.txt:
Привет
Это очень длинная строка для теста
АБВ
Средняя строка
X

Длины строк:
"Привет" = 6 символов
"Это очень длинная строка для теста" = 35 символов
"АБВ" = 3 символа
"Средняя строка" = 14 символов
"X" = 1 символ

Самая короткая: "X" (1 символ)
Самая длинная: "Это очень длинная строка для теста" (35 символов)

Результат в task8_result.txt:
САМАЯ КОРОТКАЯ СТРОКА:
X

САМАЯ ДЛИННАЯ СТРОКА:
Это очень длинная строка для теста
```
✓ Тест пройден

**Тест 8.2: Файл с одинаковыми по длине строками**
